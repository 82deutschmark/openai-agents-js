Cloudflare Workers 以及其他 workerd 运行时无法使用全局 `WebSocket` 构造函数发起向外的 WebSocket 连接。为便于在这些环境中连接实时智能体（Realtime Agents），extensions 包提供了一个专用传输器，在内部通过基于 `fetch()` 的升级流程完成连接。

<Aside type="caution">
  此适配器仍处于测试阶段。可能会遇到边缘情况或错误。 请通过 [GitHub
  Issues](https://github.com/openai/openai-agents-js/issues)
  报告问题，我们会尽快修复。对于在 Workers 中使用 Node.js 风格的 API，建议启用
  `nodejs_compat`。
</Aside>

## 设置

<Steps>

1. **安装 extensions 包。**

   ```bash
   npm install @openai/agents-extensions
   ```

2. **创建传输器并将其附加到您的会话。**

   <Code lang="typescript" code={cloudflareBasicExample} />

3. **连接您的 `RealtimeSession`。**

   ```typescript
   await session.connect({ apiKey: 'your-openai-ephemeral-or-server-key' });
   ```

</Steps>

## 注意事项

- Cloudflare 传输器在底层使用带有 `Upgrade: websocket` 的 `fetch()`，并跳过等待套接字的 `open` 事件，以匹配 workerd 的 API。
- 使用此传输器时，`RealtimeSession` 的所有特性（工具、护栏等）都可照常工作。
- 使用 `DEBUG=openai-agents*` 在开发过程中查看详细日志。

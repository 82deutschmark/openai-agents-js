“上下文”是一个多义词。通常有两大类别的上下文需要关注：

1. **本地上下文**：你的代码在一次运行期间可访问的内容，如工具所需的依赖或数据、`onHandoff` 等回调，以及生命周期钩子。
2. **智能体/LLM 上下文**：语言模型在生成回复时可见的内容。

## 本地上下文

本地上下文由 `RunContext<T>` 类型表示。你可以创建任意对象来保存状态或依赖，并将其传入 `Runner.run()`。所有工具调用和钩子都会接收一个 `RunContext` 包装器，从而可以读取或修改该对象。

参与同一次运行的每个智能体、工具和钩子都必须使用相同的上下文**类型**。

本地上下文的使用场景包括：

- 关于本次运行的数据（用户名、ID 等）
- 依赖，例如日志器或数据获取器
- 辅助函数

<Aside type="note">
  上下文对象**不会**发送给 LLM。它是纯本地的，你可以自由地读取或写入。
</Aside>

## 智能体/LLM 上下文

当调用 LLM 时，它唯一可见的数据来自于对话历史。要提供更多信息，你可以：

1. 将其添加到智能体的 `instructions` —— 也称为系统消息或开发者消息。它可以是静态字符串，也可以是接收上下文并返回字符串的函数。
2. 在调用 `Runner.run()` 时将其包含在 `input` 中。这与 instructions 的方式类似，但可让你把消息放在[指挥链](https://cdn.openai.com/spec/model-spec-2024-05-08.html#follow-the-chain-of-command)的较低位置。
3. 通过函数工具暴露它，使 LLM 可按需获取数据。
4. 使用检索或 Web 搜索工具，将回复锚定在来自文件、数据库或网络的相关数据上。

## 支持的环境

OpenAI Agents SDK 支持以下服务器环境：

- Node.js 22+
- Deno 2.35+
- Bun 1.2.5+

### 限制支持

- **Cloudflare Workers**：Agents SDK 可用于 Cloudflare Workers，但当前存在一些限制：
  - SDK 目前需要启用 `nodejs_compat`
  - 需要在请求结束时手动刷新追踪数据。详情请参阅[追踪指南](/openai-agents-js/zh/guides/tracing#export-loop-lifecycle)。
  - 由于 Cloudflare Workers 对 `AsyncLocalStorage` 的支持有限，部分追踪可能不准确
  - 出站 WebSocket 连接必须使用基于 fetch 的升级（不能使用全局 `WebSocket` 构造函数）。用于 Realtime 时，请在 `@openai/agents-extensions` 中使用 Cloudflare 传输（`CloudflareRealtimeTransportLayer`）。
- **浏览器**：
  - 浏览器当前不支持追踪
- **v8 隔离环境**：
  - 如果使用带有合适浏览器 polyfills 的打包器，应该可以为 v8 隔离环境打包 SDK，但追踪功能将不可用
  - v8 隔离环境尚未进行广泛测试

## 调试日志

如果您在使用 SDK 时遇到问题，可以启用调试日志以获取更多信息。

通过将环境变量 `DEBUG` 设置为 `openai-agents:*` 来启用调试日志。

```bash
DEBUG=openai-agents:*
```

或者，可以将调试范围限定到 SDK 的特定部分：

- `openai-agents:core` — SDK 的主要执行逻辑
- `openai-agents:openai` — OpenAI API 调用
- `openai-agents:realtime` — 实时智能体相关组件
